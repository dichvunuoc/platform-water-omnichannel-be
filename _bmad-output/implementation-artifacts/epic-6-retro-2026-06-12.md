## Thách thức

1. **`useCache` signature documentation error (Story 6.1)** — Spec ghi sai chữ ký `portRegistry.execute()`. Bắt trong implementation, không phải review. Nguyên nhân: copy pattern mà không verify chữ ký thực tế.
2. **Session event stubs (Stories 6.2, 6.3)** — `[SESSION EVENT STUB]` thay vì Redis writes thực. Intentional debt — test verify logger call, không verify Redis persistence. Epic 7 sẽ thay thế.
3. **Cross-module story complexity (Story 6.2)** — Chạm 3 modules (communication, payment, ticket), tăng blast radius.

## Bài học chính

1. **Loose coupling là tài sản thực** — Origin modules (Payment, Ticket) chỉ dispatch command, không biết channel hay preference. Communication module xử lý rest.
2. **Intentional tech debt là công cụ chiến lược** — Stubs được đánh dấu rõ, bounded, gán đúng epic. Tránh accidental debt (như signature mismatch).
3. **Pattern consistency tự củng cố** — Mỗi story dễ hơn khi previous story thiết lập convention. 'Previous Story Learnings' hoạt động.
4. **Epic 7 yêu cầu mindset shift** — Từ CQRS pattern provision sang low-level system primitives (Redis Lua, raw HTTP body, HMAC).
